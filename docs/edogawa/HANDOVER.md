## 実装ノート（このリポジトリでの実際の判断）

指示書は「build.py に江戸川ステップを追加」「全国系テーブルを共用」を前提にしていたが、
着手時点のリポジトリの実態に合わせて次のとおり実装した（指示書§0「既存8場と同じ構造で」に従った結果）。

- **取得方式＝鳴門・戸田と同じ差分蓄積**（build.py の毎朝365日全取得ではなく、cursor.txt 以降の
  差分だけ取得）。build.py は丸亀専用で cron 22:00 固定であり、そこへ相乗りすると
  「時刻オフセットを競合させない」（§9）と矛盾するため、独立スクリプト＋独立ワークフローにした。
- **全国F持ち表・1コース地力表は自前生成**。指示書が共用するとした
  `全国_F持ち_当期.csv` / `全国_1コース地力表_202607更新.csv` はリポジトリに存在しなかった
  （各会場が `data/<会場>/` 配下に自前で持つのが実態）。同じKファイルから江戸川窓で生成する。
- **配置**：当地4表＋全国2表は `data/edogawa/`、ページは `edogawa/index.html`、
  スクリプトは `scripts/edogawa/`、ワークフローは独立した `.github/workflows/edogawa.yml`。
- **ナビ**：`venues.js` に江戸川が既登録済みだったため、追加作業は不要だった。
- **cron**：既存が 22:00/22:00/22:20/22:30/22:40/22:50 のため、空き枠 **22:10 UTC（JST 7:10）** を使用。
- **オッズ**：`scripts/odds.py` の正規表現は既に `class="oddsPoint[^"]*"` で正しく、
  `data/odds/03/` の検品でも 1レース=120通り・欠落ゼロを確認済み（§9の懸念は解消済み）。

## 4. 崩れ筋スコア（江戸川版）

### 4-1. 採点式

| 項目 | 加点 | 由来 |
|---|---|---|
| **⑨ 初日（節の第1日）** | **+2** | 江戸川固有・最重要 |
| **⑭ 前半戦ゾーン（2〜6R）** | **+1** | 江戸川固有・採用 |
| **⑭' 6R（上記に上乗せ）** | **+1**（計+2） | 江戸川固有・暫定フラグ |
| ① 1号艇のスローST > .18 | +1 | 丸亀共通式 |
| ⑩ 1号艇 F1 +1 / F2以上 +2 | +1〜2 | 丸亀共通式 |
| ④ 3 or 4号艇が絞りまくり型 | +1 | 丸亀共通式 |
| ⑫ 2号艇が壁弱（素通し型） | +1 | 丸亀共通式 |

- **⑨だけ +2 なのは意図的**（初日イン残差−10.6ptは丸亀⑨の約2倍）。2日目以降は加点しない。
- **⑭はゾーンが主・6Rが従**。2〜6Rに一律+1、6Rだけさらに+1（6Rの合計は+2）。
- **⑭と⑨は独立に加算**（初日−5.9pt＋ゾーン−3.3pt ≒ 重ね掛け−9.4pt）。
  → **初日の6R は ⑨+2 と ⑭+2 で計 +4** が江戸川の最濃条件。
- **1R・9R には加点しない**（WFで減衰／符号反転＝棄却済み）。

### 4-3. 採点に使わないもの

- **風は採点しない**（朝7時のビルド時点でレース時の風は不明＝事後変数）。直前確認欄に表示のみ。
- **潮・まくり関連は一切採点しない**（§7で棄却）。

## 5. 買い目フォーム（江戸川版）

処理順＝ **①日目・番号でフォーム決定 → ②4表＋全国表をregnoでJOINして選手加点 → ③方針テキスト生成**。

| 状況 | 表示テキストの向き |
|---|---|
| 初日（単独） | 「初日＝イン最弱日（実測40.5%）。頭≠1を主軸」 |
| 初日 × 前半戦 | 「初日の前半戦＝実測32.5%。頭≠1を本線に」 |
| 初日 × 6R | 「初日の6R＝最も1号艇が飛ぶ組合せ。頭≠1を本線に」（最濃条件） |
| 6R（2日目以降） | 「6R＝最もインが飛ぶ番号（実測34.9%）。頭≠1を主軸」 |
| 初日 × ⑥飛ぶ型の1号艇 | 「1総外し検討」 |
| 初日 × ⑥残す型の1号艇 | 「X-1-Y 主体（配当は伸びない点に注意）」 |
| 通常 | 中立 |

- 6コース3連率27.1%（全国最高）＝**6は常にヒモに厚く**置く土台フォーム（全レース共通で表示）。
- **負の選別を優先**：残し表の艇番6_3連対率 **< 25%** の選手が6号艇なら「6ヒモ外し●（外し）」。
- **6コース残りは日目で動かない**（初日24.6%≒4日目以降26.4%）。「初日だから6が来る」は誤り。

## 7. 実装してはいけないもの（検証で棄却済み）

## 10. 既知の限界

- 江戸川の当地エッジは「初日」＋「前半戦（特に6R）」の2軸。潮・まくりが棄却されたため丸亀より構成は薄い。
- **江戸川では未バックテスト**（暫定運用）。識別ツールであって利益ツールではない。
- EV検証は `data/odds/03/` が1年貯まってから。最優先＝初日の1号艇が過小に嫌われているか。
