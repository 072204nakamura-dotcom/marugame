根拠：仕様書§2①（第2版）・企画分離検証（東京ベイランチ z=+3.12/+3.54、企画vs通常 z=+3.03）

## 0. なぜ「レース番号」でなく「レース名」で判定するのか

平和島の堅い/崩れるは、レース番号ではなく**番組編成**が決めている。

- 東京ベイランチ（1号艇A級・他B級）は**季節でレース番号が移動する**：夏季（薄暮開催・実測5〜8月）は2R、他季（実測9〜4月）は5R。
- 番号で判定すると夏に反転する：5Rは「他季+10.2pt（鉄板）→夏−14.0pt（崩れ）」、2Rは「他季−16.9pt→夏+4.8pt」。
- レース名で判定すれば、**移動を自動吸収**でき、季節ロジックが一切不要になる。

分離検証の決定打（同じレース番号の中で企画と通常を比べた結果）：

| セル | 残差 | イン率 | n |
|---|---|---|---|
| 5R×東京ベイランチ | +13.4pt | 78.5% | 65 |
| 5R×通常戦 | −8.3pt | 41.7% | 96 |
| 2R×東京ベイランチ | +13.6pt | 80.0% | 35 |
| 2R×通常戦 | −14.1pt | 32.5% | 126 |

## 1. 入力とレース名の実表記

- 入力：Bファイル（出走表）またはKファイルのレース名フィールド。
- **必ずNFKC正規化してから判定する**（全角英数・全角スペース対策。既存パイプラインの共通処理）。
- Kファイルのレース名は**6文字程度に切詰められる**。365日窓の実表記（全23種）で確認済みの要注意表記：

| 実表記 | n | 正体 |
|---|---|---|
| `東京ベイラン` | 100 | 東京ベイランチ（企画）。「チ」が切れる |
| `ベイブレイク` | 99 | ベイブレイク（企画） |
| `東京ベイ選抜` | 3 | 企画ではない選抜戦。「東京ベイ」で拾うと誤爆する |
| `東京ベイＤＲ` | 1 | 企画ではない（ドリーム系）。同上 |
| `トーキョーベ` | 1 | 切詰め表記。企画ではない扱い（フォールバック） |
| `準優勝戦` | 64 | 「優勝」を含むので判定順序に注意 |

→ 企画判定パターンは「東京ベイ」ではなく **「ベイラン」**（誤爆ゼロを実測確認）。ベイブレイクは **「ベイブレ」**。

## 2. 判定ロジック（優先順位つき）

**上から順に評価し、最初に一致した分類で確定する**（順序を守らないと「準優勝戦」が「優勝」に誤一致する）。

| 順 | パターン（部分一致） | 分類 | フラグ／加点 | 根拠（残差・イン率・万舟率・n） |
|---|---|---|---|---|
| 1 | `ベイラン` | 東京ベイランチ | **鉄板フラグ（穴買い禁止）** | +13.5pt・78〜80%・12〜17%・n=100 |
| 2 | `ベイブレ` | ベイブレイク | **+1点＋カド攻めフォーム** | −10.4pt・46.5%・14.1%・n=99 |
| 3 | `準優` | 準優勝戦 | 中立（加点なし） | −1.4pt・62.5%・n=64 |
| 4 | `優勝` | 優勝戦 | 鉄板寄り表示（穴買い注意） | +11.6pt・76.7%・n=30 |
| 5 | `記者選抜` | 記者選抜 | 中立〜やや堅（加点なし） | +3.3pt・69.9%・n=73 |
| 6 | `選抜` | 選抜系（平和島選抜・特別選抜等） | **参考+0.5点（隠れ穴・暫定）** | −13.4pt・49.4%・**万舟23.0%**・n=87 ※z未検定の暫定。EV検証キュー入り |
| 7 | （どれにも一致しない） | 通常戦 | → レース番号点へ進む | — |

**レース番号点（分類7＝通常戦のみに適用）**：

| 条件 | 加点 | 根拠 |
|---|---|---|
| 7R | +2点 | −18.5pt・z=−3.23・通年安定（夏−17.1/他季−19.5） |
| 3R | +1点 | −16.1pt・z=−2.51・通年安定 |
| 1R・6R | +0.5点 | 方向のみ（単独非有意） |

**廃止した判定**：5R加点・12R加点（レース番号ベースの鉄板判定は全廃）。

## 3. 買い目フォームの分岐

企画判定は点数だけでなく**フォーム（買い目の型）**も切り替える：

- **東京ベイランチ**：フォーム＝本命（1アタマ固定）。ツールの主旨（穴）から外れるため「見送り推奨」表示。
- **ベイブレイク**：フォーム＝**カド攻め**。1&4号艇A級編成なので、崩れ筋は「4カドまくり → 4-1・4-流し」。差し水面フォーム（2C/3Cアタマ）は適用しない。壁表との重ね掛け：**2枠に壁弱●なら4まくりが刺さりやすい**。
- **通常戦の崩れゾーン（7R/3R）**：フォーム＝**差し水面**〈2C/3Cアタマ−1号艇2着残し−広め〉。2コース差し表の差し巧者●が2枠なら「2-1本線」を明示。
- **優勝戦**：1アタマ基本・見送り寄り。

## 4. 参照実装（Python・そのまま貼り付け可）

プログラミング未経験でも追えるよう、1行ずつ何をしているかコメントを付けています。

```python
import unicodedata

def judge_heiwajima_race(rname, race_no):
    """
    平和島のレース分類と加点を返す。
    rname : レース名（B/Kファイルから読んだ文字列）
    race_no : レース番号（1〜12の整数）
    戻り値 : (分類名, 加点, フラグ, フォーム) の4つ
    """
    # 手順1: 全角/半角のゆれを吸収する（例: 全角英数字→半角）。
    #        既存パイプラインと同じNFKC正規化。
    name = unicodedata.normalize('NFKC', rname or '')

    # 手順2: 優先順位の高い順に部分一致で調べる。
    #        「in」は「この文字列が含まれているか」の判定。
    if 'ベイラン' in name:
        # 東京ベイランチ（切詰め表記「東京ベイラン」を含む）
        return ('東京ベイランチ', 0, '鉄板・穴買い禁止', '見送り推奨')
    if 'ベイブレ' in name:
        # ベイブレイク（1&4号艇A級 → カドが強い）
        return ('ベイブレイク', 1.0, 'カド攻め', '4まくり(4-1/4-流し)')
    if '準優' in name:
        # 「準優勝戦」。次の『優勝』より先に調べないと誤一致する。
        return ('準優勝戦', 0, '', '中立')
    if '優勝' in name:
        return ('優勝戦', 0, '鉄板寄り', '1アタマ・見送り寄り')
    if '記者選抜' in name:
        return ('記者選抜', 0, '', '中立')
    if '選抜' in name:
        # 平和島選抜・特別選抜など。隠れ穴（暫定）。
        return ('選抜系', 0.5, '暫定・隠れ穴', '差し水面フォーム')

    # 手順3: どの企画にも当てはまらない＝通常戦。
    #        ここで初めてレース番号の点を使う。
    if race_no == 7:
        return ('通常戦', 2.0, '主戦場', '差し水面フォーム(2C/3Cアタマ-1残し)')
    if race_no == 3:
        return ('通常戦', 1.0, '準主戦場', '差し水面フォーム(2C/3Cアタマ-1残し)')
    if race_no in (1, 6):
        return ('通常戦', 0.5, '参考', '差し水面フォーム')

    # 手順4: それ以外の通常戦は加点なし。
    return ('通常戦', 0, '', '中立')
```

racecard.py への組み込み位置：**6表JOINより前**。
処理順＝ ①この関数で分類・基礎点・フォームを決定 → ②6表（ST/壁/残し/絞りまくり/2コース差し/F持ち）をregnoでJOINして選手加点 → ③合計点と買い目方針テキストを出力。

## 6. 保守メモ

- 平和島が企画名を改称・新設した場合（過去例：Afternoon Pleasant Race→東京ベイランチ、2023年ベイブレイク新設）、**未知のレース名は自動で通常戦に落ちる**設計なので誤爆はしないが、鉄板検出が漏れる。月次で `heiwajima_races.csv` のレース名一覧（§1の表の再集計）を眺め、新名称が出たらパターンに追加する。
- 選抜系+0.5点は**暫定**（n=87・z未検定）。三条件ルールの正式検定とEV検証を通るまで参考扱いを維持。
- 他場への展開：この「レース名優先→番号フォールバック」構造は、企画が番号移動する場（多摩川・蒲郡等）へ将来展開する際のテンプレートになる。
